# 项目开发日志 (Project Journal)

## 缘起与宏观目标
本项目（代号：Nexus402 / wrap402）的诞生，源于我们对于 AI 时代一个明确趋势的判断：**未来的互联网数据消费者不仅仅是人类，还会存在海量的智能体（AI Agents）进行自主的数据获取与服务调用**。

目前的互联网商业架构建立在针对“人类用户”的包月订阅、扫码支付上。而 AI 则受限于身份验证、注册壁垒，常常面对 403 (Forbidden) 或登录墙束手无策。
为此，我们要重置一套对机器友好的计费标准——基于 HTTP `402 Payment Required` 协议的数据代理网关。

## 主要对话与实施脉络

### Phase 1: 概念论证与核心 CLI 的构建
- 我们讨论并确定了 `wrap402` 这个终端工具的核心定位。它是这个项目的基石，通过读取本地配置 `.wrap402.json` 将本地的数据（最初设想是简单的 JSON 静态资产）转变成带计费的 HTTP API。
- 确立了 HTTP 402 的拦截流程：用户/AI 访问资源 `->` 被截获抛出 402 `->` 提示先完成支付签名获取 Token `->` 注入 Header 重试 `->` 返回数据。
- 引入了 `/.well-known/x402-assets.json` 作为 AI 爬虫进入网站时的默认寻址入口。

### Phase 2: 生态扩展——加入反向代理 (Reverse Proxy) 功能
- **决策制定**：如果只支持本地静态数据，这个平台的使用场景会非常狭窄。我们决定加入对“动态 API”的转发支付能力。
- **技术实施**：在 Node 后端利用类似代理透传的方案。比如通过 Nexus402 的公开端口 `4020`，我们为其设定了背后对应真实的失信人员全网接口或货币汇率 API。
- **成果**：完成从纯“静态货架”进化为“计费网关”的重要一步。

### Phase 3: GUI 面板 (Web Dashboard) 开发
- 我们确定要面向没有深厚技术背景的数据服务商。因此需要一个具有赛博朋克极客风格的亮眼深色 Dashboard。
- 选型决定使用 `Vite + React + Tailwind + Lucide` 的现代组合，抛弃庞大臃肿的框架，追求极速响应。
- 实现了能在前后端分离下，利用对后端的 REST 请求，直接在网页上实现新数据/API的上架、定价和无缝生效的过程。

### Phase 4: Financial Risk Control Agent 演示
- 为了验证 Nexus402 这个产品在现实世界的绝对可行性，我们在项目中构建了实际业务模拟：**小额贷款信用审查**。
- **演示剧本**：我们扮演由第三方出资的“信贷审批 AI”；它需要从“借贷记录局”和“最高法失信黑名单库”获取资料。虽然这些属于外部收费机构，但由于它们部署了 Nexus402 节点，我们的 AI 通过脚本自动完成了 402 协议感知、账单支付和核心绝密档案的下载，最终做出不予放贷的金融决策。
- 这一步直接印证了产品的底层价值逻辑，至此，MVP 最小可行性产品已经完全闭环。

### Phase 5: 云端化与文档整理
- 项目推进到了代码进入 GitHub 版本控制的阶段。
- 我们补充了完整的业务层说明：《Business Plan》(商业计划书) 和《Architecture》(产品架构设计)，彻底将我们的灵感不仅沉淀在代码层面，也定格在了宏伟的发展蓝图中。

### Phase 6: 账户系统、Webhook 与智能网关
- **账户管理 (`accounts.ts`)**：实现了完整的开发者账户系统，包含余额管理、充值 (`topup`)、消费 (`spend`)、API Key 生成与轮转等功能。账户数据持久化到本地 `.wrap402-accounts.json` 文件。
- **Webhook 异步通知 (`webhooks.ts`)**：实现了带指数退避重试重试逻辑的 Webhook 服务，在支付成功、资产创建等事件时自动推送通知至开发者配置的回调地址。
- **Smart Agent Gateway (`/api/v1/agent/ask`)**：构建了一个自然语言驱动的智能网关。AI Agent 只需用"人话"描述需求，Gateway 会自动搜索最匹配的资产、完成账户扣款、获取数据并返回结构化结果。这使得 Agent 无需了解具体的资产 ID 或 API 路径。
- **Dashboard 扩展**：新增"账户余额"Tab（API Key 管理、Webhook 配置、余额充值）和"Agent 实验室"Tab（可视化测试智能网关）。

### Phase 7: TypeScript SDK 与 Agent 演示矩阵
- **TypeScript SDK (`sdk/typescript/`)**：将 x402 交互协议封装为开发者友好的 SDK 类 `Nexus402Client`，提供 `discover()` 资产发现、`request()` 自动 402 处理/支付/重试、`ask()` 自然语言网关调用、`getBalance()` 余额查询等方法。
- **Market Agent 演示 (`examples/market-agent.ts`)**：基于 SDK 构建的行业趋势分析 Agent，展示了完整的 Discover → Select → Pay → Access 四步闭环流程。
- **Dashboard 扩展**：新增"数据分析"Tab，包含 7 天营收趋势图表（Recharts AreaChart）、调用频次可视化、头部资产营收贡献率看板。
